strerror与perror

  1. 功能区别
  • strerror：
    • 返回一个错误码对应的描述性字符串。
    • 需要程序员显式打印返回的字符串。
    • 常用于需要自定义错误信息格式或进一步处理错误字符串的场景。
  • perror：
    • 打印错误信息到标准错误流（`stderr ），格式为：[用户自定义前缀]: 错误消息。
    • 不返回字符串，仅输出信息。
    • 更适合简单的错误信息打印场景。
  1. 使用场景区别
  • strerror：
    • 更灵活，返回一个描述性字符串，便于组合输出、记录日志等。
    • 适合需要对错误字符串做进一步处理的场景。
  • perror：
    • 更简洁，直接将错误信息打印到标准错误流。
    • 适合快速输出错误信息并调试的场景。
  1. 参数区别
  • strerror(int errnum)：
    • 接收一个错误码（errno）作为参数，返回对应的描述性字符串。
    • 不会自动打印，需显式使用 printf 或其他方式输出。
  • perror(const char *message)：
    • 接收一个用户自定义的字符串作为前缀，将其与当前 errno 的错误信息一起打印到标准错误流。
    • 不返回任何值，直接输出。

    总结的表格如下：

    特性 strerror perror
    功能 返回错误信息字符串 打印错误信息到标准错误流 (stderr)
    参数 接收错误码 (errno) 作为参数 接收一个用户自定义的字符串作为前缀
    输出方式 需要程序员手动打印返回值 自动将错误信息打印到 stderr
    灵活性 灵活，适合自定义错误信息或进一步处理错误字符串 简洁，直接用于快速调试和标准输出
    适用场景 用于自定义错误消息格式和日志记录 用于快速调试错误
    返回值 返回指向错误描述的字符串 无返回值，直接输出
    使用示例 printf("Error: %s\n", strerror(errno)); perror("Error");
    输出格式 自定义格式（由用户决定输出内容和样式） 格式固定，例如：Error: No such file or directory

下面介绍一下snprintf函数：
snprintf 是 C 语言中的一个标准库函数，用于将格式化的数据写入字符串中。它的基本功能和 sprintf 类似，但是 snprintf 提供了一个额外的安全特性：它可以指定目标缓冲区的最大大小，从而避免写入超出缓冲区的内容，减少了缓冲区溢出的风险
函数原型：

参数说明:

  • str：指向存储格式化输出的目标字符串的指针。
  • size：目标字符串的最大大小（包括终止的空字符 \0）。
  • format ：格式化字符串，类似于printf中的格式字符串。
  • ...：根据格式字符串提供的其他参数。
    返回值:
  • snprintf 返回实际写入的字符数（不包括终止的空字符）。如果返回值大于或等于 size，则说明目标缓冲区大小不足，字符串没有完全写入。
    特性:
  • 如果写入的数据超出了 size - 1 个字符（即，目标缓冲区不能容纳完全格式化的字符串），则 snprintf 会截断数据，只写入最多 size - 1 个字符，并在末尾添加一个空字符 \0 作为字符串终止符。
  • 如果写入的数据不超过 size - 1 个字符，snprintf 会正常写入并以空字符结束字符串。
  • snprintf 比 sprintf 更安全，因为它避免了缓冲区溢出的风险。
    使用举例：
    解释:
  • buffer 是存储输出字符串的目标数组，大小为 50。
  • sizeof(buffer) 返回 50，表示目标缓冲区的最大容量。
  • 格式化字符串 “Number: %d, Pi: %.2f” 用于输出一个整数和一个浮点数。
  • snprintf 将格式化后的字符串写入 buffer，并返回实际写入的字符数（包括终止符 \0）。
    重要注意点:
  1. 缓冲区大小：如果 size 参数不足以容纳完整的输出字符串，snprintf 会截断字符串，并确保字符串以 \0 结束。因此，在使用时，确保 size 至少是目标缓冲区的大小。
  2. 返回值检查：返回值大于等于 size 时，表示输出被截断，可以检查该值来判断是否发生了截断。
  3. 线程安全：snprintf 是线程安全的。
    总结:
    snprintf 是一个安全的字符串格式化函数，它可以防止缓冲区溢出，适用于格式化并写入字符串的场景。使用时需要确保目标缓冲区足够大，以容纳格式化后的内容。
